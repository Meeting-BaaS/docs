## Prerequisites

- Python ^3.11
- `grpc_tools` for protocol buffer compilation
- Ngrok (for local deployment)
- Poetry version ^1.8.3 for dependency management

## Installation

### 1. Set Up Poetry Environment

```bash
# Install Poetry (Unix/macOS)
curl -sSL https://install.python-poetry.org | python3 -

# Install Poetry (Windows)
(Invoke-WebRequest -Uri https://install.python-poetry.org -UseBasicParsing).Content | py -

# Install dependencies
poetry install

# Activate virtual environment
poetry shell
```

### 2. Compile Protocol Buffers

```bash
poetry run python -m grpc_tools.protoc --proto_path=./protobufs --python_out=./protobufs frames.proto
```

Protocol Buffers are used here by Pipecat to define a structured message format for real-time communication between components of the Speaking Bots system. Specifically, the [`frames.proto`](https://github.com/pipecat-ai/pipecat/blob/635aa6eb5bdee382729613b58279befdc5bc8eaf/src/pipecat/frames/frames.proto#L9) file defines three main message types:

1. `TextFrame`: For handling text-based messages
2. `AudioRawFrame`: For managing raw audio data with properties like sample rate and channels
3. `TranscriptionFrame`: For handling speech-to-text transcription results

Protocol Buffers is here the backbone of consistent data serialization across services. Read more in the [official Protocol Buffer documentation](https://protobuf.dev/downloads/) and [this Python Protocol Buffers tutorial](https://www.blog.pythonlibrary.org/2023/08/30/an-intro-to-protocol-buffers-with-python/).

### 3. Configure Environment

```bash
cp env.example .env
```

Edit `.env` with the required credentials:

```bash
# MeetingBaas API Key - Used for sending meeting bots as personas
MEETING_BAAS_API_KEY=your_meetingbaas_api_key_here

# OpenAI API Key - Used for in-meeting AI interactions
OPENAI_API_KEY=your_openai_api_key_here

# Speech-to-Text (choose ONE of the following):
DEEPGRAM_API_KEY=your_deepgram_api_key_here
# OR
GLADIA_API_KEY=your_gladia_api_key_here

# Text-to-Speech
CARTESIA_API_KEY=your_cartesia_api_key_here

# Ngrok Auth Token - Required for local development and automated CLI
NGROK_AUTHTOKEN=your_ngrok_auth_token_here
```

These are the core credentials needed to run a single Speaking Bot. Additional variables are required for multiple bots or persona management features.

For more information on ngrok configuration and how the multi-bot setup works, see our [ngrok configuration guide](/docs/speaking-bots/ngrok-setup).
